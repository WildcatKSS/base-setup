# Contributing

## Core Rule

No change is accepted unless it improves customer value, validation, profitability, security, maintainability, or operational reliability.

## Required Before Work

- Read CLAUDE.md
- Check project-status.md
- Check scope-control.md
- Check relevant ADRs
- Check relevant acceptance criteria

## Pull Request Requirements

Every PR must include:
- Business justification
- Related documentation
- Test evidence
- Security impact
- Deployment impact
- Rollback notes

## Architecture Changes

Any significant architecture change requires an ADR before implementation.

## Product Changes

Any feature change must link to:
- PRD
- Customer problem
- Acceptance criteria
- Scope-control decision
