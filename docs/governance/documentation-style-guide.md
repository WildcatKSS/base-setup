# Documentation Style Guide

## Heading Hierarchy

### Level 1
Use exactly one `#` heading for the document title.

### Level 2
Use `##` for major sections.

### Level 3
Use `###` for subsections.

### Level 4
Use `####` only when necessary.

Never go deeper than level 4.

## File Naming

Use kebab-case for all files.

Examples:

- project-status.md
- docs/context/current-state.md
- decision-framework.md

## Writing Rules

- Keep sections short.
- Prefer lists over long paragraphs.
- Avoid duplicate governance rules.
- Link to the canonical source document.


## Document References

Always use full repository-relative paths.

Good:
- `docs/context/current-state.md`

Avoid:
- `docs/context/current-state.md`
