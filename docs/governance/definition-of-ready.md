# Definition of Ready

Work may start only when the item is ready.

## Business Readiness

- Customer problem is known.
- Business justification is documented.
- Expected value is clear.
- MVP relevance is known.
- Unit economics impact is understood where relevant.

## Product Readiness

- User story or requirement exists.
- Acceptance criteria are defined.
- Scope decision is made: Must, Should, Could or Won't.
- Explicitly out-of-scope items are documented.

## Technical Readiness

- Architecture impact is understood.
- Security impact is understood.
- Dependencies are known.
- Lean Engineering ladder has been applied.
- Existing functionality has been checked for reuse or deletion.

## Delivery Readiness

- Verification approach is known.
- Documentation updates are identified.
- Rollback or failure path is understood for production-impacting changes.

If any section is unclear, the work is not ready.
