# Phase Gates

## Purpose

Phase Gates prevent Claude Code from skipping essential validation, governance and quality checks.

## Gates

## Phase 0 – Governance Foundation

Required:
- `CLAUDE.md` exists.
- Governance documents exist.
- Context memory files exist.
- ADR structure exists.
- Lean Engineering exists.

## Phase 1 – Market Discovery

Required:
- At least 10 B2B SaaS opportunities considered.
- Opportunities scored.
- Assumptions marked clearly.
- No product code written.

## Phase 2 – Business Validation

Required:
- Validation plan exists.
- Customer interview script exists.
- Competitor analysis exists.
- Pricing hypothesis exists.
- Unit economics hypothesis exists.

## Phase 3 – PRD

Required:
- PRD complete.
- MoSCoW priorities defined.
- MVP scope explicit.
- Won't Have section explicit.
- Acceptance criteria defined.

## Phase 4 – Architecture

Required:
- Architecture documented.
- Security requirements documented.
- ADRs created for major decisions.
- Observability plan documented.
- Cost implications considered.

## Phase 5 – GitHub Foundation

Required:
- PR template exists.
- Issue templates exist.
- CI/security workflow skeletons exist.
- Dependabot exists where applicable.

## Phase 5A – MVP Gate

Required:
- MVP solves one core problem.
- Customer can pay.
- Scope is minimal.
- Unit economics likely positive.
- First 10 customer path is clear.
- Security/compliance risk acceptable.

## Phase 6 – Implementation

Required:
- MVP Gate approved.
- Definition of Ready satisfied.
- Lean Engineering applied.
- Tests or checks added with non-trivial logic.

## Phase 7 – Testing

Required:
- Critical flows covered.
- Payment/auth/core workflow tested where applicable.
- Security checks performed.

## Phase 8 – DevOps & Operations

Required:
- Deployment path documented.
- Rollback path documented.
- Monitoring/error tracking planned.
- Secrets management documented.

## Phase 9 – Launch

Required:
- Launch checklist complete.
- First 10 customer plan exists.
- Support path exists.
- Feedback loop exists.

## Phase 10 – Continuous Governance

Required:
- Metrics reviewed.
- Roadmap updated from evidence.
- Unit economics reviewed.
- Context files updated.

## Rule

No phase may be skipped. Moving to the next phase requires explicit approval.
